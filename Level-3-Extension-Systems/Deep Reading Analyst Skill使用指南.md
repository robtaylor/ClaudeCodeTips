# Deep Reading Analyst Skill 使用指南

## 📖 Skill 概述

**Deep Reading Analyst** 是一个系统化的深度阅读分析框架，通过多种思维模型（结构化思维、第一性原理、批判性思维、系统思维、六顶思考帽）将表层阅读转化为深度学习，帮助用户从理解到应用的完整知识转化过程。

### 核心能力

- ✅ 深度理解复杂文章/论文内容
- ✅ 分析论证质量并识别逻辑漏洞
- ✅ 从阅读材料中提取可执行洞察
- ✅ 创建学习笔记和知识总结
- ✅ 对比多个信息源
- ✅ 将知识转化为实际应用

### 适用场景

| 场景 | 触发关键词 | 典型用法 |
|------|-----------|---------|
| 文章分析 | "分析这篇文章" | 理解技术博客、论文 |
| 深度理解 | "帮我深入理解" | 学习复杂概念 |
| 洞察提取 | "提取洞察" | 从案例中学习经验 |
| 知识整理 | "总结要点" | 创建学习笔记 |
| 多源对比 | "对比观点" | 综合不同来源信息 |

## 🎯 快速开始

### 基本使用

```bash
# 直接提供 URL 进行分析
帮我深度分析这篇文章：https://example.com/article

# 或者复制文本内容
请分析以下文章内容：
[粘贴文章文本]
```

### 交互式配置

Skill 会询问三个关键问题：

1. **阅读目标**：问题解决 / 学习总结 / 写作参考 / 纯粹好奇
2. **分析深度**：Quick / Standard / Deep / Research
3. **关注焦点**：特定部分或主题（可选）

**如果不回复，默认使用 Level 2 (Standard) 模式。**

## 📊 四级分析深度

### Level 1: Quick (15分钟)

**适用场景**：快速了解文章核心观点

**输出内容**：
- 文章结构图
- TOP 3 关键洞察
- 1 个立即行动项

**使用示例**：
```
快速总结这篇文章的核心观点
```

### Level 2: Standard (30分钟) ⭐推荐

**适用场景**：标准深度分析，平衡效率与质量

**输出内容**：
- 结构化理解
- 批判性思维分析
  - 论证质量评估
  - 逻辑漏洞识别
  - 证据可信度评价
  - 替代视角探索

**使用示例**：
```
深度分析这篇 Claude Code 最佳实践文章
```

### Level 3: Deep (60分钟)

**适用场景**：复杂内容的全面分析

**输出内容**：
- Standard 级别所有内容
- 第一性原理分析（本质提取）
- 系统思维分析（关系映射）
- 六顶思考帽（多视角审视）

**使用示例**：
```
需要非常深入地分析这篇论文，包括多角度思考
```

### Level 4: Research (120分钟+)

**适用场景**：学术研究或跨来源综合分析

**输出内容**：
- Deep 级别所有内容
- 跨来源对比（自动搜索相关文章）
- 观点对比矩阵
- 综合评价

**使用示例**：
```
研究级别分析：对比多个关于 AI 辅助编程的观点
```

## 🔍 核心分析框架

### 1. 结构化理解（所有级别必含）

```markdown
📄 内容类型: [文章/论文/报告/指南]
⏱️ 阅读时间: [X 分钟]
🎯 核心论点: [一句话概括]

结构:
├─ 主要论点 1
│   ├─ 支撑点 1.1
│   └─ 支撑点 1.2
├─ 主要论点 2
└─ 主要论点 3

关键概念: [3-5 个术语]
高价值章节: [段落/章节编号]
```

### 2. 批判性思维（Level 2+）

**证据评估矩阵**：

| 证据类型 | 强度 | 警示信号 |
|---------|------|---------|
| 同行评审研究 | ★★★★★ | 样本量、利益冲突 |
| 原始数据 | ★★★★★ | 收集方法、偏差 |
| 专家共识 | ★★★★☆ | 领域共识 vs 单一专家 |
| 案例研究 | ★★★☆☆ | 选择偏差、可推广性 |
| 轶事证据 | ★★☆☆☆ | 不具代表性 |

**逻辑谬误检查清单**：
- [ ] 因果谬误（相关性≠因果性）
- [ ] 证据谬误（樱桃采摘、幸存者偏差）
- [ ] 修辞谬误（人身攻击、诉诸权威、稻草人）
- [ ] 统计谬误（忽略基准率、误导性平均值）

### 3. 系统思维（Level 3+）

识别文章内容中的：
- **反馈环路**：正反馈和负反馈机制
- **涌现特性**：整体>部分之和
- **杠杆点**：小改动大影响的关键节点
- **延迟效应**：因果之间的时间滞后

### 4. 多视角分析（Level 3+）

**六顶思考帽**：
- 🔴 红帽（情感）：直觉和感受
- ⚪ 白帽（事实）：客观数据
- 🟡 黄帽（乐观）：优势和价值
- ⚫ 黑帽（批判）：风险和缺陷
- 🟢 绿帽（创新）：替代方案
- 🔵 蓝帽（控制）：整体评估

## 💡 实际应用案例

### 案例 1：分析技术博客

**输入**：
```
帮我深度分析这篇文章：
https://www.dzombak.com/blog/2025/08/getting-good-results-from-claude-code/

目标：学习最佳实践并应用到我的项目
```

**Skill 行为**：
1. 使用 Playwright 获取完整文章内容
2. 提取作者的 4 大核心实践（SPEC.md、CLAUDE.md、Self-review、Global guide）
3. 批判性分析：识别幸存者偏差（只展示成功项目）
4. 提取可执行洞察：3 次尝试规则、分层文档体系
5. 生成立即行动项：创建项目的 SPEC.md

**输出价值**：
- ✅ 15 分钟内完成原本需要 1 小时的深度阅读
- ✅ 获得结构化笔记而非零散印象
- ✅ 明确 24 小时内可执行的行动项

### 案例 2：对比多篇文章

**输入**：
```
对比分析以下三篇关于 Prompt Engineering 的文章，
找出共识和分歧点：
1. [URL 1]
2. [URL 2]
3. [URL 3]
```

**Skill 行为**：
1. 分别分析三篇文章（Level 2）
2. 构建对比矩阵
3. 识别核心共识（如"清晰性"）
4. 标注分歧点（如"详细度 vs 简洁性"）
5. 提供综合建议

### 案例 3：学习复杂概念

**输入**：
```
我不太理解 MCP (Model Context Protocol)，
帮我分析这篇官方文档并整理学习笔记
```

**Skill 行为**：
1. 提取核心概念（Server、Client、Transport）
2. 构建概念关系图
3. 用类比简化理解（如"MCP = API for AI"）
4. 生成验证性问题测试理解
5. 推荐实践项目

   - 论证质量评估

   - 证据可信度分析

   - 从基本真理重建理解

   - 识别反馈循环
   - 寻找杠杆点
   - 理解涌现特性

   - 多角度审视问题
   - 平衡情感与理性
   - 发现创新可能

## ✅ 质量标准

每次分析保证：

- ✅ 忠实于原始内容（无歪曲）
- ✅ 区分事实与观点
- ✅ 提供具体示例
- ✅ 连接用户的具体场景
- ✅ 以可执行步骤结尾

避免：

- ❌ 一次性使用所有框架（overwhelming）
- ❌ 无解释的学术术语
- ❌ 只分析不给应用建议
- ❌ 逐字复制原文（总是重新表述）

## 🎯 最佳实践

### 1. 明确你的目标

在请求分析前，清楚回答：
- 我为什么要读这篇文章？
- 我希望从中获得什么？
- 我会如何使用这些信息？

### 2. 选择合适的深度

| 情况 | 推荐级别 |
|-----|---------|
| 快速决策是否深入阅读 | Level 1 |
| 日常学习和工作应用 | Level 2 ⭐ |
| 复杂主题系统学习 | Level 3 |
| 学术研究或写作 | Level 4 |

### 3. 主动参与分析

Skill 会提出引导性问题，积极回答能获得更个性化的分析：
- "你认为作者说 X 是什么意思？"
- "你看到这个论证的漏洞了吗？"
- "你会如何在工作中应用这个？"

### 4. 立即行动

不要只收藏分析结果，在 24 小时内尝试一个"快速赢单"行动项。

## 🆘 常见问题

**Q: 和直接问 Claude "总结这篇文章"有什么区别？**

A:
- 普通总结：提取关键点（表层信息压缩）
- Deep Reading Analyst：批判性分析 + 可执行洞察 + 知识激活（深度理解转化）

**Q: 每次分析都需要回答三个问题吗？**

A: 不是。如果你不回答，Skill 会使用默认的 Level 2（Standard）模式和学习目标。

**Q: 可以分析中文文章吗？**

A: 完全可以。Skill 支持中英文内容分析。

**Q: 分析一篇文章需要消耗多少 tokens？**

A: 取决于文章长度和分析深度：
- Level 1: ~2K-5K tokens
- Level 2: ~5K-15K tokens
- Level 3: ~15K-30K tokens
- Level 4: ~30K-50K tokens

**Q: 可以保存分析结果吗？**

A: 分析结果以 Markdown 格式输出，你可以：
1. 直接复制到笔记工具（Obsidian、Notion）
2. 保存为本地 .md 文件
3. 导出为 PDF
